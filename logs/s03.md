## Objective: Understanding functions, parameters, and repetition in Python

### Defining Functions
- Learned the structure of function definitions: `def` keyword, function name, parentheses, colon, and indented body
- Understood that the header contains the function name and parameters, and the body contains the executable statements
- Discovered that function names follow the same rules as variable names
- Realized that defining a function creates a function object that can be inspected

### Parameters and Arguments
- Learned that parameters are variable names in the function definition that receive argument values
- Understood that when a function is called, the argument value is assigned to the parameter
- Discovered that variables can be used as arguments, not just literal values
- Realized that functions can take multiple parameters

### Calling Functions
- Learned that functions can call other functions, creating a chain of calls
- Understood that when multiple functions work together, they execute in a specific order
- Realized that one function can be reused inside multiple other functions
- Discovered that this modularity makes code more organized and maintainable

### The `for` Loop for Repetition
- Learned the structure: `for` keyword, variable name, `in`, and a sequence like `range(n)`
- Understood that `range(n)` creates a sequence of numbers from 0 to n-1
- Discovered that the loop body runs once for each value in the sequence
- Realized that `for` loops can be combined with functions for powerful repetition patterns

### Local Variables and Parameters
- Learned that variables created inside a function are **local** and only exist within that function
- Understood that when a function ends, its local variables are destroyed
- Discovered that trying to access a local variable outside its function causes a `NameError`
- Realized that parameters are also local to their function

### Stack Diagrams and Tracebacks
- Learned that stack diagrams show which variables belong to which functions at each call level
- Understood that each function is represented as a **frame** containing its parameters and local variables
- Discovered that tracebacks display the call chain when an error occurs
- Realized that this helps identify where errors happen in complex function calls

### Why Functions Matter
- Learned that functions let you name groups of statements for better readability
- Understood that functions eliminate repetitive code and allow changes in one place
- Discovered that functions enable debugging one piece at a time
- Realized that well-designed functions can be reused across many programs

### Debugging Insights
- Learned that debugging is similar to detective work and experimental science
- Understood the importance of taking small steps and testing frequently
- Discovered that spending too much time debugging often signals writing too much code before testing
- Realized that incremental development catches errors early
