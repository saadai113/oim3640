# Learning Log - Session 06
## Objective: Understanding conditionals, recursion, and boolean logic in Python

### Integer Division and Modulus
- Learned the modulus operator `%`` which returns the remainder after division
- Discovered that `x % 10`` extracts the rightmost digit of a number
- Understood how modulus works with "clock arithmetic" (e.g., `(11 + 3) % 12 = 2`)
- Realized modulus is useful for checking divisibility: if `x % y == 0`, then `x` is divisible by `y`

### Boolean Expressions
- Learned the equals operator `==` compares two values and returns `True` or `False`
- Key mistake: don't confuse `=` (assignment) with `==` (comparison)
- Understood that `True` and `False` are special values of type `bool`, not strings
- Explored relational operators: `==`, `!=`, `<`, `>`, `<=`, `>=`

### Logical Operators
- Learned the three main logical operators: `and`, `or`, and `not`
- `and` returns `True` only if both conditions are true
- `or` returns `True` if at least one condition is true
- 'not' negates a boolean expression
- Discovered Python treats any nonzero number as 'True' and zero as 'False'

### if Statements
- Understood the structure: 'if' keyword, condition, colon, indented block
- Learned the condition must be a boolean expression
- If the condition is true, the indented block runs; otherwise it's skipped
- Met the 'pass' statement, which does nothing but serves as a placeholder for future code

### Key Insights
- Boolean logic forms the foundation for controlling program flow
- Conditionals allow programs to make decisions based on conditions
- The modulus operator is more powerful than it first appears
