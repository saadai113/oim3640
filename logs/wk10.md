# Objective: Building the final project for resume parsing and job matching
 - Worked on the final project that parses resumes and matches them to job descriptions
 - Learned how to extract text from PDF, DOCX, TXT, RTF, and HTML files
 - Used OpenAI structured outputs with Pydantic so the resume parser returns clean JSON
 - Learned how to keep API keys in environment variables with `load_dotenv`
 - Worked on both the command-line pipeline and the Flask app interface
 - Added matching features that rank candidates against a job description
 - Learned how to rewrite resumes so they better fit a job description without inventing new experience
 - Biggest challenge was handling different file formats and keeping the output structured and reliable

## Resume Parser
 - Learned how `resume_parser.py` extracts text from resumes and sends it to the OpenAI API
 - Used a Pydantic schema to define fields like name, email, work experience, education, skills, and years of experience
 - Understood the importance of telling the model not to invent information
 - Learned that OCR can be used as a fallback for image-based PDFs

## Matching and Rewriting
 - Learned how `match_llm.py` scores a resume directly against a job description
 - Learned how `match_embeddings.py` uses embeddings for faster first-pass ranking
 - Understood why embeddings are cheaper and more scalable than scoring every resume with the LLM
 - Learned how `resume_rewriter.py` improves the wording of a resume while preserving the original facts
 - Realized that the rewrite step should only rephrase existing content, not fabricate achievements

## Flask App
 - Worked on `app.py`, which provides routes for matching resumes and rewriting them through a web app
 - Learned how to upload multiple resumes through a form and process them on the backend
 - Learned how to return JSON results and sort candidates by overall match score
 - Learned how to create a Stripe checkout session for paid scanning and rewriting credits
 - Realized that the app needs good error handling for missing files, invalid file types, and API failures

## Key Takeaways
 - The project showed how LLMs can be used as typed infrastructure instead of just chatbots
 - Learned that clean text extraction is important before sending resumes to embeddings or the LLM
 - Understood that cost and scalability matter when processing many resumes
 - Learned that structured outputs and schema validation make the pipeline much more reliable
